# Purpose

Defines the expected frontmatter shape for product category records.

# Required fields

`id`, `type`, `title`, `status`, `retrieval_priority`, `reviewed_at`.

# Notes

category-first fit, examples, avoid-when notes, trust and evidence routes. Health-related records must also include `safety_boundary` and `prohibited_claims`.
