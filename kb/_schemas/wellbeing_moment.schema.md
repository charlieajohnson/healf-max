# Purpose

Defines the expected frontmatter shape for wellbeing moment records.

# Required fields

`id`, `type`, `title`, `status`, `retrieval_priority`, `reviewed_at`.

# Notes

Pre-modelled customer situation joining signals, safety and next-step planning. Health-related records must also include `safety_boundary` and `prohibited_claims`.

Useful route fields: `evidence_routes`, `product_lanes`, `editorial_signals`, `tone_patterns`, and `trust_signals`.
